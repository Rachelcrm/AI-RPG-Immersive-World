# Incident Report: Database Performance Degradation

## Incident Summary

- **Date**: January 15, 2023
- **Duration**: 3 hours 45 minutes (09:15 - 13:00 UTC)
- **Impact**: Increased API response times (5x normal), 15% of user requests timed out
- **Services Affected**: User Authentication, Product Catalog, Order Processing
- **Severity**: High

## Timeline

- **09:15 UTC**: First alerts triggered for increased API latency
- **09:22 UTC**: On-call engineer acknowledged alert
- **09:35 UTC**: Identified database as the bottleneck (high query times)
- **10:10 UTC**: Determined root cause as a missing index on a frequently queried column
- **11:30 UTC**: Applied index to production database
- **12:45 UTC**: Performance returned to normal levels
- **13:00 UTC**: Incident declared resolved

## Root Cause

A deployment the previous day included a new query pattern that frequently filtered on an unindexed column in the `product_inventory` table. As traffic increased during business hours, this led to full table scans instead of index seeks, causing database CPU usage to spike to 95% and significantly increasing query response times.

## Detection

The incident was detected through our monitoring system when:
1. API response time alerts triggered (threshold: p95 > 500ms for 5 minutes)
2. Database CPU utilization exceeded 90% for more than 10 minutes
3. Error rate for timeout exceptions increased above 5%

## Resolution

1. Added the missing index on the `product_inventory.last_updated` column
2. Optimized the problematic query to use existing indexes where possible
3. Increased the connection pool size to handle the backlog of requests

## Lessons Learned

### What Went Well
- Monitoring quickly detected the performance degradation
- The on-call engineer was able to identify the database as the bottleneck within 20 minutes
- Adding the index resolved the issue without requiring a service restart

### What Went Wrong
- The deployment process did not catch the missing index requirement
- Database query performance was not tested under load before deployment
- The incident took too long to resolve due to initial troubleshooting in the wrong areas

## Action Items

1. Add database query analysis to the CI/CD pipeline to catch missing indexes
2. Implement automatic index suggestion based on query patterns
3. Create a runbook for database performance issues
4. Add more detailed database metrics to monitoring dashboards
5. Schedule regular database performance reviews

## Metrics to Monitor

Based on this incident, we should closely monitor:
- Database CPU utilization
- Slow query counts and patterns
- Table scan vs. index seek ratios
- Connection pool utilization
- Query timeout rates
